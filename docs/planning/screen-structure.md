# 화면 구조

## 주요 화면

| 화면 | 역할 |
|---|---|
| 시작 안내 | 프로젝트 목적과 체험 방법 안내 |
| 이동약자 유형 선택 | P-00~P-03 체험 조건 선택 |
| 앱 도움 선택 | 앱 OFF / ON 조건 선택 |
| 3D 체험 화면 | 이동, 신호, 장애물, 앱 안내 경험 |
| 결과 화면 | 이동 기록과 앱 요구사항 확인 |
| 설정 화면 | 시야 효과, TTS, 화면 흔들림 등 조정 |

## 3D 체험 화면 구성

- 중앙: 3D 이동 환경
- 상단: 현재 조건과 진행 상태
- 왼쪽: 진행 체크 카드
- 오른쪽: 상태 UI와 신호 정보
- 하단: 이동 보조 앱 안내 패널

## 학생용 화면 원칙

- 개발자 좌표와 디버그 정보는 노출하지 않는다.
- 강제 선택지보다 실제 이동 행동을 우선한다.
- 성공/실패 점수보다 관찰 기록을 보여준다.
