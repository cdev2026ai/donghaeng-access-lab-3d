# 사용자 플로우

## 기본 흐름

```text
접속
→ 체험 목적 확인
→ 이동약자 유형 선택
→ 앱 도움 OFF/ON 선택
→ 연속 경로 체험
→ 관찰 결과 확인
→ 앱 요구사항 정리
```

## 권장 체험 흐름

1. 앱 도움 OFF 상태로 먼저 체험한다.
2. 이동 중 어떤 정보가 부족했는지 관찰한다.
3. 결과 화면에서 부족했던 정보를 확인한다.
4. 같은 유형으로 앱 도움 ON 상태를 다시 체험한다.
5. 두 조건의 차이를 앱 요구사항으로 정리한다.

## 화면 중단 원칙

학생용 버전은 장애물마다 선택지를 띄우는 미션형 구조를 지양한다. 이동 흐름은 최대한 연속적으로 유지하며, 안내는 하단 앱 패널과 상태 UI 중심으로 제공한다.
