# v0.9.1 신호등·고령 보행 테스트 체크리스트

## A. 신호등 오브젝트

- [ ] 횡단보도 접근 방향에서 신호등 표시판이 보인다.
- [ ] 적색 신호에서 정지 사람 모양이 붉게 표시된다.
- [ ] 녹색 신호에서 보행 사람 모양이 녹색으로 표시된다.
- [ ] 신호등 본체에 남은 시간이 숫자로 표시된다.
- [ ] 녹색 시간은 17초부터 연속적으로 감소한다.
- [ ] 종료 임박 5초 동안 녹색 보행 표시가 점멸한다.
- [ ] HUD와 3D 신호등 본체의 상태·시간이 일치한다.
- [ ] 시점을 조금 바꾸어도 표시판이 사라지지 않는다.

## B. 신호 주기와 차량

- [ ] RED_WAIT 8초 후 보행 신호가 시작된다.
- [ ] GREEN_START 1초, GREEN_ACTIVE 11초, FLASH_WARNING 5초가 순환한다.
- [ ] 보행 가능 시간 동안 차량이 정지한다.
- [ ] 횡단 중 신호가 종료되면 차량 정지가 유지된다.
- [ ] 횡단자가 없는 적색 보행 신호에서 차량이 다시 움직인다.

## C. 고령 보행자 횡단

- [ ] P-02의 일반 이동 속도는 기존과 동일하게 55%다.
- [ ] P-02가 횡단보도에 들어가면 이동 속도가 추가로 낮아진다.
- [ ] 녹색 시작 후 약 1.5초간 출발 반응 지연이 적용된다.
- [ ] 즉시 출발해도 17초 안에 반대편 도착이 어렵다.
- [ ] 신호 종료 후 횡단 중에도 차량은 안전 정지한다.
- [ ] 결과 화면에 `신호 종료 전: 아니오` 또는 이에 준하는 기록이 표시된다.
- [ ] 시간 압박과 불안감이 횡단 중 증가한다.

## D. 이동 보조 앱 안내

- [ ] 앱 OFF에서는 고령 보행 조건을 사용자가 직접 관찰한다.
- [ ] 앱 ON에서는 `현재 신호 시간만으로 횡단이 어렵습니다` 안내가 표시된다.
- [ ] 앱 ON에서 신호 연장 또는 보행 지원 필요가 제안된다.
- [ ] 단순히 다음 신호를 기다리라는 안내만 반복하지 않는다.

## E. 회귀 테스트

- [ ] P-00은 17초 안에 횡단할 수 있다.
- [ ] P-01은 경사로 정렬 후 17초 안에 횡단할 수 있다.
- [ ] P-03은 앱·음향 조건에 따라 신호 정보 접근 차이가 유지된다.
- [ ] 장애물 5종, 연속 체험, 결과 화면이 기존처럼 작동한다.
- [ ] TypeScript 검사와 Vite 빌드가 통과한다.
