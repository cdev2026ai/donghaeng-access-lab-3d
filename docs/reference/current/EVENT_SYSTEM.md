# 행동 기반 이벤트 시스템 v0.9

## 변경 원칙

v0.4의 `접근 → 상황인지 화면 → 판단 선택 → 수행 → 개별 결과` 모달 상태 머신은 기본 학생 체험에서 사용하지 않습니다. v0.9에서는 이벤트를 화면 인터럽트가 아니라 연속 이동 중 발생하는 행동·환경 로그로 처리합니다.

## 경로 상태

```text
READY
→ SIDEWALK_ENTRY
→ WAIT_SIGNAL
→ CROSS_PREP
→ CROSSING
→ POST_CROSS
→ ARRIVED
```

## 장애물 상태

각 장애물은 다음 데이터를 내부적으로 기록합니다.

- 접근 여부와 접근 시각
- 근접 체류 시간
- 정지 횟수
- 충돌·이동 차단
- 경로 재조정
- 통과 시각

## 이동 메커니즘 해석

- 상황인지: 시점 회전, 정지, 신호·환경 확인, F 탐지, 앱 안내
- 상황판단: 기다림, 출발, 다음 신호 선택, 우회·경사로 접근
- 동작수행: 이동 속도, 정렬, 통과 경로, 충돌, 후진·재진입

## 사용자 인터럽트

자동 장애물 모달은 없습니다. 다음 화면만 사용자가 요청하거나 명확한 단계 전환 때 표시합니다.

- 시작·유형·앱 조건·체험 안내
- 일시정지·설정·도움말·분석
- 목적지 도착 결과

## 분석 모드

분석 패널은 기존 단계 구조를 강사 설명용으로 유지합니다.

- 현재 상황인지 정보
- 현재 판단 상태
- 현재 동작수행 상태
- 신호·예상 횡단·시간 여유
- 이동 보조 앱 I-P-O
