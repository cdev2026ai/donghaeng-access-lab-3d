# v0.9 수업용 테스트 체크리스트

## A. 시작 흐름

- [ ] 처음 접속 시 체험 목적 안내가 가장 먼저 보인다.
- [ ] 유형을 선택하기 전 3D 이동이 시작되지 않는다.
- [ ] 유형 선택 후 앱 도움 없음/있음 조건을 선택할 수 있다.
- [ ] 짧은 체험 안내 후에만 포인터 잠금과 이동이 시작된다.

## B. 연속 체험

- [ ] 장애물 접근 시 전체 화면 판단 선택 창이 자동으로 열리지 않는다.
- [ ] 학교 정문부터 버스정류장까지 한 번의 연속 이동으로 체험할 수 있다.
- [ ] 장애물 접근, 정지, 우회, 차단, 통과가 자동 기록된다.
- [ ] ESC·설정·도움말·분석은 사용자 요청 때만 열린다.

## C. 횡단보도·신호

- [ ] 보행 적색·녹색·종료 임박·전 방향 정지 상태가 순환한다.
- [ ] 보행 가능 신호에서 차량이 정지선 앞에 멈춘다.
- [ ] 적색 보행 신호에서 차량이 다시 움직인다.
- [ ] 위험 진입 시 차량 비상 정지와 위험 진입 기록이 남는다.
- [ ] 대기 시간, 출발 지연, 출발 시 남은 시간, 횡단 시간이 기록된다.
- [ ] 신호 종료 전 횡단 완료 여부가 결과에 표시된다.

## D. 유형별 제약

- [ ] P-00은 기준 속도와 기준 시점으로 이동한다.
- [ ] P-01은 단차에서 막히며 경사로 정렬이 필요하다.
- [ ] P-02는 출발과 횡단이 느리고 시간 압박·피로가 더 쉽게 증가한다.
- [ ] P-03은 앱 OFF·음향신호 OFF에서 정확한 신호 정보를 바로 알 수 없다.
- [ ] P-03의 F 지팡이 탐지와 시야·방향 효과가 작동한다.

## E. 이동 보조 앱

- [ ] 앱 OFF에서는 비모달 앱 패널이 숨겨진다.
- [ ] 앱 ON에서는 신호·거리·방향·예상 시간·권고가 표시된다.
- [ ] TTS ON에서 우선순위가 높은 안내가 음성으로 출력된다.
- [ ] TTS OFF에서 음성이 출력되지 않는다.
- [ ] 동일 안내가 프레임마다 반복 재생되지 않는다.
- [ ] 경사로·통과 폭·우회 방향 안내가 유형별로 다르다.

## F. 결과 화면

- [ ] `미션 성공`, `안전 판단 5/5` 같은 점수형 표현이 없다.
- [ ] 관찰형 횡단·이동 기록이 표시된다.
- [ ] 앱 OFF에서는 부족했던 정보가 정리된다.
- [ ] 앱 ON에서는 앱이 제공한 정보와 개선점이 정리된다.
- [ ] 앱 도움 상태를 바꾸어 같은 유형으로 다시 체험할 수 있다.
- [ ] 성찰 입력, 누적 저장, 교차 비교, JSON·CSV 내보내기가 없다.

## G. 배포·접근성

- [ ] Chrome 또는 Edge에서 WASD·마우스·F·ESC가 작동한다.
- [ ] 1366×768 및 1920×1080에서 HUD와 모달이 잘리지 않는다.
- [ ] 저사양 모드에서 텍스트 안내와 핵심 신호 기능은 유지된다.
- [ ] GitHub Actions의 build와 deploy가 모두 성공한다.
