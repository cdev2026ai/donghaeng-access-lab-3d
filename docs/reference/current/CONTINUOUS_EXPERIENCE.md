# 연속 체험 설계 v0.9

## 목적

장애물마다 선택 창을 열어 정답을 고르게 하는 게임형 구조를 줄이고, 실제 이동처럼 한 경로를 연속해서 체험하도록 구성합니다.

## 기본 원칙

1. 전체 화면 인터럽트는 체험 시작, 사용자가 요청한 일시정지·설정·도움말·분석, 목적지 도착에만 사용합니다.
2. 장애물 접근 안내는 HUD, 비모달 앱 패널, 선택적 TTS와 환경 반응으로 제공합니다.
3. 상황인지–상황판단–동작수행은 버튼 선택이 아니라 실제 행동에서 추론하고 기록합니다.
4. 성공·실패 점수보다 대기·정지·출발·우회·경로 재조정·차단 기록을 보여줍니다.

## 행동 기록 매핑

| 이동 메커니즘 | 관찰 행동 |
|---|---|
| 상황인지 | 시점 회전, 정지, 신호 확인, F 지팡이 탐지, 앱 안내 확인 |
| 상황판단 | 신호 대기, 출발 시점, 다음 신호 선택, 우회·경사로 접근 |
| 동작수행 | 이동 속도, 경로, 경사로 정렬, 충돌, 차단, 후진·재진입 |

## 연속 경로 상태

```text
READY
→ SIDEWALK_ENTRY
→ WAIT_SIGNAL
→ CROSS_PREP
→ CROSSING
→ POST_CROSS
→ ARRIVED
```

## 인터럽트 정책

### 자동으로 열지 않는 화면

- 장애물 설명 모달
- 상황인지 확인 버튼
- 1·2·3 판단 선택지
- 장애물별 결과 모달

### 필요한 경우에만 여는 화면

- 시작 안내
- 유형 선택
- 앱 도움 조건 선택
- 짧은 체험 안내
- 사용자가 요청한 설정·도움말·분석
- 최종 관찰 결과

## 학생 체험과 분석 모드

- 학생 기본 체험: 연속 이동과 앱 정보 요구사항 발견 중심
- 분석 모드: 강사가 이동 메커니즘, 신호 상태, 앱 I-P-O를 설명할 때 사용
