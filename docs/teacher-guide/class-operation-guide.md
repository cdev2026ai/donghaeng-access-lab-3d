# 강사용 수업 운영 가이드

## 수업에서의 역할

동행 Access Lab 3D는 학생 프로젝트의 최종 산출물이 아니라, 이동 보조 앱 개발 전에 문제를 발견하는 체험 도구입니다.

강사는 학생들이 “장애물을 피하는 게임”으로 받아들이지 않도록 다음 관점을 계속 안내합니다.

> 이 체험의 목표는 빠른 도착이 아니라, 이동 중 부족한 정보를 찾아 앱 요구사항으로 바꾸는 것입니다.

## 수업 전 준비

- 학생용 URL 접속 확인
- Chrome 또는 Edge 실행 확인
- 키보드·마우스 조작 확인
- 네트워크 및 GitHub Pages 접속 확인
- 개발자용 URL은 강사용 검수에만 사용

## 추천 수업 흐름

| 단계 | 활동 | 시간 예시 |
|---|---|---|
| 1 | 프로젝트 목표 안내 | 5분 |
| 2 | 비교 기준 보행자 체험 | 5분 |
| 3 | 이동약자 유형별 체험 | 15분 |
| 4 | 앱 OFF/ON 비교 | 10분 |
| 5 | 결과 화면 관찰 | 10분 |
| 6 | 워크북 기록 및 팀 토의 | 15분 |

## 추천 진행 방식

1. 먼저 P-00 비교 기준 보행자로 경로를 짧게 시연합니다.
2. 학생 팀별로 P-01, P-02, P-03 중 하나를 선택하게 합니다.
3. 반드시 앱 도움 없음으로 먼저 체험하게 합니다.
4. 같은 유형으로 앱 도움 있음을 켜고 재체험하게 합니다.
5. 결과 화면의 “부족했던 정보”와 “앱이 제공할 정보”를 워크북으로 옮기게 합니다.

## 강사용 질문 예시

- 어느 지점에서 가장 먼저 멈췄나요?
- 앱이 없을 때 어떤 정보가 부족했나요?
- 앱이 켜졌을 때 어떤 판단이 쉬워졌나요?
- 이 기능을 실제 앱에서 구현하려면 어떤 입력 데이터가 필요할까요?
- 화면 안내와 음성 안내 중 어떤 방식이 더 적절할까요?

## 주의할 점

- 실제 장애인의 경험 전체를 재현한다고 설명하지 않습니다.
- 특정 유형이 “더 못한다”는 식으로 비교하지 않습니다.
- 결과 기록은 점수가 아니라 환경과 정보 접근성의 차이를 보여주는 자료입니다.
- 개발자용 모드는 학생 체험 중 공개하지 않는 것을 권장합니다.
