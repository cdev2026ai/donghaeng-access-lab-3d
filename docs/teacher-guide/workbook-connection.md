# 워크북 연결 가이드

## 연결 원칙

시뮬레이터는 학생이 직접 체험한 내용을 워크북의 문제 정의와 앱 기능 설계로 연결하기 위한 도구입니다.

```text
시뮬레이터 체험
→ 관찰한 어려움
→ 부족했던 정보
→ 앱 요구사항
→ 팀 프로젝트 기능
```

## 화면과 워크북 연결표

| 시뮬레이터 요소 | 학생 관찰 질문 | 워크북 기록 항목 | 앱 기능 예시 |
|---|---|---|---|
| 보도 턱 | 어디서 막혔는가? | 이동 차단 지점 | 경사로 위치 안내 |
| 가파른 경사 | 이동 속도와 피로가 어떻게 변했는가? | 이동 부담 | 경사도·주의 안내 |
| 보행 신호 | 출발해도 되는지 판단하기 쉬웠는가? | 신호 정보 | 잔여 시간·예상 횡단 안내 |
| 불법주차 차량 | 통로 폭을 알 수 있었는가? | 통로 제약 | 통과 가능 여부·우회 안내 |
| 점자블록 훼손 | 방향을 계속 유지할 수 있었는가? | 정보 단절 | 다음 유도선 위치 안내 |
| 볼라드·입간판 | 목적지 진입이 쉬웠는가? | 목적지 앞 장벽 | 진입 경로 안내 |

## 팀 활동 질문

- 우리 팀 앱은 어떤 이동약자 유형을 우선 지원할 것인가?
- 가장 먼저 해결할 이동 장벽은 무엇인가?
- 앱이 입력받아야 할 데이터는 무엇인가?
- 앱이 판단해야 할 조건은 무엇인가?
- 사용자에게 어떤 방식으로 안내할 것인가?

## 산출물 예시

```text
기능명: 보행 신호 기반 횡단 가능 안내
대상: 고령 보행자
입력: 현재 위치, 보행 신호 잔여 시간, 사용자 평균 속도, 횡단보도 길이
판단: 예상 횡단 시간이 잔여 시간보다 긴지 비교
출력: 출발 가능 / 다음 신호 대기 / 보행 지원 필요 안내
```
