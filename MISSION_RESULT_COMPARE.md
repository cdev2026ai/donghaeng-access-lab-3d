# 미션·결과·비교 화면 설계 — v0.6

## 1. 목적

v0.6은 3D 이동 자체보다 체험 전·중·후의 학습 흐름을 명확하게 만드는 단계다.

1. 체험 전: 선택한 이동 조건과 미션 목표를 확인한다.
2. 체험 중: 이벤트·안전 판단·목적지 진행 상태를 확인한다.
3. 체험 후: 이동 기록과 상태 변화를 해석하고 성찰을 작성한다.
4. 교차 체험: 같은 환경에서 서로 다른 이동 조건의 결과를 나란히 비교한다.

비교 화면은 사람의 능력을 평가하거나 순위를 정하지 않는다. 환경 장벽과 이동 조건의 상호작용을 관찰하기 위한 교육 자료다.

## 2. 미션 브리핑

- 경로: 학교 정문 → 보도 → 횡단보도 → 버스정류장
- 목표: 활성 장애물 확인, 상황인지·판단 기록, 직접 동작수행, 목적지 도착
- 선택 유형 요약: 이름, 핵심 병목, 기본 속도
- 유형별 관찰 포인트 표시
- 미션 시작 전에는 장애물 이벤트가 실행되지 않는다.

## 3. 실시간 미션 UI

- 이벤트 완료 수
- 안전 중심 판단 횟수
- 목적지 도착 여부
- 기존 경로 진행률, 장애물 진행, 상태 UI와 함께 표시

## 4. 결과 화면

### 달성 상태

- 경로 도착
- 이벤트 완료
- 안전 판단 수
- 충돌·이동 차단을 합친 재시도 부담

### 기록

- 소요 시간, 이동 거리, 장애물 통과
- 충돌, 이동 차단, 거친 노면 체류
- 지팡이 탐지, 방향 편차
- 평균 판단 시간, 재확인 수
- 최종 피로도, 최대 불안감, 최저 방향 확신도, 최대 시간 압박
- 장애물별 이벤트 로그

### 성찰

- 가장 어려웠던 순간
- 어려움을 키운 환경 조건
- 포용적 환경 개선 제안

결과와 성찰은 현재 브라우저의 localStorage에 최대 8회까지 저장한다.

## 5. 비교 화면

- 최근 완료 기록 2~4개 선택
- 유형, 핵심 병목, 시간, 거리, 충돌·차단, 판단, 상태 지표 비교
- 선택한 결과 범위를 기반으로 비교 관찰 문구 생성
- 비교 CSV 내보내기
- 저장 기록 전체 삭제

## 6. 내보내기

- 개별 결과 JSON
- 개별 결과 CSV
- 선택 결과 비교 CSV
- 개인정보·계정·서버 전송 없음

## 7. 교육적 원칙

- 빠른 시간이나 낮은 상태 수치를 우열로 표현하지 않는다.
- 결과를 이동 조건과 환경 장벽의 상호작용으로 해석한다.
- 성찰 질문은 개인의 결함보다 환경 개선 방향을 찾도록 구성한다.
