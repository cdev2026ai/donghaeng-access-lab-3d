# v0.4 상황인지–판단–수행 이벤트 시스템

## 1. 목적

장애물에 단순히 부딪히거나 지나가는 방식에서 벗어나, 각 장애물을 다음의 학습 흐름으로 체험하도록 구성한다.

1. 접근(Approach)
2. 상황인지(Sensing)
3. 판단(Decision)
4. 동작수행(Action)
5. 결과(Result)

각 이벤트는 이동약자 유형, 장애물, 사용자가 선택한 판단, 실제 수행 중 충돌·이동 차단을 함께 기록한다.

## 2. 이벤트 시작 조건

- 활성화된 장애물의 감지 반경에 진입하면 이벤트 시작
- 이벤트가 시작되면 3D 이동을 잠시 멈추고 이벤트 화면 표시
- 이미 통과한 장애물과 설정에서 끈 장애물은 이벤트를 시작하지 않음
- 한 번에 하나의 이벤트만 진행

## 3. 단계별 동작

### 접근

- 장애물 이름과 이동약자 유형별 접근 문구 표시
- `상황 확인하기` 또는 `Enter`로 다음 단계 이동

### 상황인지

- 장애물에서 확인해야 하는 높이·폭·경사·바닥·방향 정보를 안내
- 비교 기준, 휠체어, 고령 보행, 시각장애 유형별 확인 방법 표시
- 시각장애 유형은 `F` 키 또는 버튼으로 흰지팡이·바닥 정보 확인
- 확인한 정보와 확인 방법 기록

### 판단

- 장애물마다 3개 선택지 제공
  - 안전 중심 선택
  - 위험 가능 선택
  - 상황 재확인 선택
- 숫자 `1`, `2`, `3` 또는 마우스로 선택
- 재확인을 선택하면 상황인지 단계로 돌아가며 재확인 횟수 기록
- 판단에 걸린 시간 기록

### 동작수행

- 선택한 판단 문구를 HUD에 표시
- 사용자가 WASD와 마우스로 직접 장애물을 통과
- 수행 중 충돌, 이동 차단, 소요 시간 기록
- 장애물 통과 지점을 지나면 결과 단계로 전환

### 결과

- 확인 방법, 판단, 수행 시간, 충돌, 차단, 재확인 횟수 표시
- 안전 판단 여부와 실제 수행 결과를 결합해 결과 문구 생성
- `다음 구간으로`를 누르면 다음 장애물 체험 재개

## 4. 결과 판정

| 판단 | 수행 결과 | 결과 문구 |
|---|---|---|
| 안전 중심 | 충돌·차단 없음 | 안전한 판단으로 충돌 없이 통과 |
| 안전 중심 | 충돌 또는 차단 발생 | 안전 경로를 선택했지만 수행 중 재시도 발생 |
| 위험 가능 | 충돌 또는 차단 발생 | 위험한 판단 뒤 충돌 또는 이동 차단 발생 |
| 위험 가능 | 충돌·차단 없음 | 직접 경로로 통과했지만 위험 요인을 남김 |

## 5. 최종 결과 화면

- 완료 이벤트 수
- 안전 판단 횟수
- 재확인 횟수
- 평균 판단 시간
- 장애물별 확인 방법
- 장애물별 선택한 판단
- 장애물별 수행 시간
- 장애물별 충돌 및 이동 차단
- 장애물별 종합 결과

## 6. 키보드 조작

| 키 | 기능 |
|---|---|
| WASD / 방향키 | 3D 이동 |
| 마우스 | 시점 회전 |
| Enter / E | 이벤트 다음 단계 |
| 1 / 2 / 3 | 판단 선택지 선택 |
| F | 시각장애 유형의 상황인지·지팡이 탐지 |
| ESC | 일시정지 |
| R | 현재 유형으로 미션 초기화 |

## 7. 교육적 해석

이 이벤트 시스템은 특정 장애를 흉내 내는 것이 아니라, 환경 정보와 이동 조건의 상호작용을 구조적으로 관찰하기 위한 교육용 모델이다. 시야 제한·피로도·불안감·방향 확신도와 같은 체험 효과는 다음 단계에서 상태 변수와 HUD로 연결한다.
