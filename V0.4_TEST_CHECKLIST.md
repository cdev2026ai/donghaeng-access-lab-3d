# v0.4.1 테스트 체크리스트

## A. 공통 실행

- [ ] `npm ci`가 오류 없이 완료된다.
- [ ] `npm run build`가 완료된다.
- [ ] 시작 화면에 `v0.4-event-system`이 표시된다.
- [ ] 이동약자 유형 4개를 선택할 수 있다.
- [ ] 장애물 5종이 기존과 동일하게 표시된다.

## B. 이벤트 공통 흐름

각 장애물 O-01~O-05에 대해 반복 확인한다.

- [ ] 감지 범위 진입 시 3D 이동이 멈춘다.
- [ ] 이벤트 화면이 `접근` 단계로 열린다.
- [ ] `상황 확인하기`를 누르면 `상황인지`로 이동한다.
- [ ] 정보 확인 전에는 판단 단계로 바로 넘어가지 않는다.
- [ ] 정보 확인 후 확인 결과가 표시된다.
- [ ] 판단 단계에서 선택지 3개가 표시된다.
- [ ] 숫자 1·2·3으로 선택할 수 있다.
- [ ] 선택 전에는 `선택 적용` 버튼이 비활성화된다.
- [ ] 재확인을 선택하면 상황인지 단계로 돌아간다.
- [ ] 안전 또는 위험 선택을 적용하면 3D 이동이 재개된다.
- [ ] 선택한 수행 문구가 하단 HUD에 표시된다.
- [ ] 장애물 통과 후 결과 화면이 열린다.
- [ ] 결과 화면에 판단·시간·충돌·차단이 표시된다.
- [ ] `다음 구간으로`를 누르면 다음 장애물로 이동할 수 있다.

## C. 이동약자 유형별

### P-00 비교 기준

- [ ] 일반적인 시각·공간 정보 확인 문구가 표시된다.
- [ ] 5개 이벤트를 순서대로 완료할 수 있다.

### P-01 수동 휠체어

- [ ] 높이·폭·경사·통과 가능성 확인 문구가 표시된다.
- [ ] O-01에서 경사 통로 우회 선택지가 표시된다.
- [ ] O-03에서 넓은 우회 경로 선택지가 표시된다.
- [ ] 단차·좁은 통로 충돌이 해당 이벤트 기록에 반영된다.

### P-02 고령 보행

- [ ] 근거리 확인·균형 점검 문구가 표시된다.
- [ ] O-02에서 잠시 멈춘 뒤 천천히 이동 선택지가 표시된다.
- [ ] 기존 반응 지연과 이동 감속이 유지된다.

### P-03 시각장애

- [ ] 상황인지 단계에서 F 키 안내가 표시된다.
- [ ] F 키로 정보 확인을 완료할 수 있다.
- [ ] 지팡이·바닥 정보 확인 횟수가 결과에 반영된다.
- [ ] 기존 점자블록 방향 편차가 유지된다.
- [ ] 지팡이 3D 모델은 아직 표시되지 않는다는 범위가 문서와 일치한다.

## D. 이벤트 기록

- [ ] 안전 선택은 안전 판단 횟수에 포함된다.
- [ ] 위험 선택은 안전 판단 횟수에 포함되지 않는다.
- [ ] 재확인 횟수가 누적된다.
- [ ] 판단 시간이 0 이상으로 기록된다.
- [ ] 수행 중 충돌 수가 장애물별 기록에 반영된다.
- [ ] 수행 중 이동 차단 수가 장애물별 기록에 반영된다.
- [ ] 최종 결과 화면에 장애물별 기록 5개가 표시된다.

## E. 초기화·설정·예외

- [ ] R 키로 이벤트 기록과 장애물 진행이 모두 초기화된다.
- [ ] 유형 변경 시 이벤트 기록이 초기화된다.
- [ ] 장애물을 끄면 완료 목표 수에서 제외된다.
- [ ] 이벤트 화면이 열린 상태에서 ESC로 잘못된 일시정지 화면이 열리지 않는다.
- [ ] 이벤트 결과를 닫기 전 최종 완료 화면이 먼저 열리지 않는다.
- [ ] 마지막 O-05 결과 확인 후 버스정류장 도착 화면이 열린다.
- [ ] 저사양 모드에서도 이벤트 UI가 정상 표시된다.

## F. 접근성·문구

- [ ] 이벤트 진행 단계가 텍스트로 표시된다.
- [ ] 선택지는 색상 외에 문구로 안전·위험·재확인을 구분한다.
- [ ] 버튼을 키보드로 조작할 수 있다.
- [ ] 이벤트 결과가 `aria-live` 영역에서 갱신된다.
- [ ] 실제 장애 경험 전체를 재현하지 않는다는 교육용 고지가 유지된다.
