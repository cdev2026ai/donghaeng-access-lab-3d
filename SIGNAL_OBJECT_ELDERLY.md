# 신호등 오브젝트·고령 보행 조건 보완 v0.9.1

## 1. 보행 신호등 오브젝트

기존 신호등은 HUD에는 상태가 표시되지만 3D 신호등 본체의 표시가 작고 방향에 따라 보이지 않을 수 있었습니다.

v0.9.1에서는 신호등 본체에 다음 요소를 표시합니다.

- 적색 정지 사람 모양
- 녹색 보행 사람 모양
- 현재 상태의 남은 시간 숫자
- 종료 임박 구간의 점멸 표시
- 켜진 상태의 보조 표시등과 발광 효과

신호 표시판은 횡단보도에 접근하는 방향에서 보이도록 배치하며, 양면 재질을 사용해 시점 변화에도 사라지지 않도록 합니다.

## 2. 보행 신호 주기

| 상태 | 시간 | 설명 |
|---|---:|---|
| INIT_SAFE | 1초 | 화면 복귀·초기화 안전 상태 |
| RED_WAIT | 8초 | 보행 대기, 차량 통행 |
| GREEN_START | 1초 | 보행 신호 시작 |
| GREEN_ACTIVE | 11초 | 일반 보행 가능 |
| FLASH_WARNING | 5초 | 종료 임박 점멸 |
| ALL_STOP | 2초 | 전 방향 안전 정지 |

보행 가능한 총시간은 17초입니다.

## 3. 고령 보행자 횡단 조건

P-02 고령 보행자·지팡이 사용자 모드는 다음 조건을 적용합니다.

- 일반 이동 속도: 기준의 55%
- 횡단보도 내 이동 속도: 일반 속도에 40% 추가 적용
- 녹색 신호 후 출발 반응 지연: 1.5초
- 횡단 정렬 지연: 0.8초

12m 횡단보도 기준 단순 이동 시간은 약 17.3초이며, 실제 출발 지연까지 포함하면 약 18.8초가 필요합니다. 따라서 현재 17초의 보행 가능 시간 안에 완전히 건너기 어렵게 구성했습니다.

이 수치는 모든 고령자의 실제 보행 능력을 일반화하는 값이 아니라, 신호 시간이 이동 조건을 충분히 고려하지 못할 때 발생하는 문제를 학생이 확인하도록 만든 교육용 시뮬레이션 조건입니다.

## 4. 이동 보조 앱 안내

앱 도움 ON에서 P-02가 횡단보도에 접근하면 다음 취지의 안내가 표시됩니다.

- 현재 신호 시간만으로 한 주기 안에 횡단하기 어려움
- 신호 연장 또는 보행 지원 필요
- 체험에서는 신호 종료 후에도 차량이 안전 정지함

이 안내는 단순히 다음 신호를 기다리라는 권고가 아니라, 이동 보조 앱이 개인의 이동 속도와 신호 시간을 비교해 인프라 지원 필요까지 알려야 한다는 요구사항을 보여 줍니다.

## 5. 안전 처리

고령 보행자가 신호 종료 후에도 횡단보도 안에 있으면 차량은 계속 안전 정지합니다. 결과 화면에는 다음 항목이 기록됩니다.

- 출발 시 남은 신호 시간
- 실제 횡단 시간
- 신호 종료 전 완료 여부
- 차량 안전 정지 여부
