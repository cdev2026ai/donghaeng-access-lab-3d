# v0.4.1 시작 순서 오류 수정

## 수정 전

앱이 실행되자마자 시작 위치가 O-01 감지 반경 안에 있어 `updateObstacleTracking()`이 실행되고, 시작 안내와 이동약자 유형 선택보다 장애물 이벤트가 먼저 표시될 수 있었습니다.

## 수정 후

1. 시작 안내 화면
2. 이동약자 유형 선택
3. 선택한 유형으로 체험 시작
4. 사용자가 약 0.15m 이상 이동
5. O-01 접근 이벤트 시작

## 코드 보호 조건

- `hasStarted === false`이면 이벤트 탐색 중단
- 미션 완료 상태이면 이벤트 탐색 중단
- 첫 이동 전에는 새 이벤트 시작 금지
- 일시정지·설정·도움말 화면 중에는 새 이벤트 시작 금지
- 초기화 시 `intro-modal`을 명시적으로 다시 엶
