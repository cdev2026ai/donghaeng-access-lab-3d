# 미션·결과 화면 설계 — v0.6.1

## 1. 설계 방향

시뮬레이터는 체험에 필요한 정보와 결과 확인까지만 담당한다.

1. 체험 전: 선택한 이동 조건과 미션 목표를 확인한다.
2. 체험 중: 이벤트·안전 판단·목적지 진행 상태를 확인한다.
3. 체험 후: 이동 기록과 상태 변화를 결과 화면에서 확인한다.
4. 성찰·교차 체험·개선안 작성은 워크북과 팀 활동으로 이어간다.

동일한 성찰 문항과 비교 활동을 시뮬레이터 안에서 다시 수행하지 않도록 기능을 단순화했다.

## 2. 미션 브리핑

- 경로: 학교 정문 → 보도 → 횡단보도 → 버스정류장
- 선택 유형: 이름, 핵심 병목, 기본 속도
- 목표: 장애물 확인, 정보·판단 확인, 직접 동작수행, 결과 확인
- 유형별 관찰 포인트 표시
- `미션 시작` 전에는 이동과 장애물 이벤트가 실행되지 않는다.

## 3. 실시간 미션 UI

- 이벤트 완료 수
- 안전 중심 판단 횟수
- 목적지 도착 여부
- 경로 진행률
- 장애물 진행 상태
- 피로도·불안감·방향 확신도·시간 압박

## 4. 결과 화면

### 미션 달성 상태

- 목적지 도착
- 이벤트 완료 수
- 안전 판단 수
- 충돌·이동 차단을 합친 재시도 부담

### 체험 기록

- 소요 시간, 이동 거리, 장애물 통과
- 충돌, 이동 차단, 거친 노면 체류
- 지팡이 탐지, 방향 편차
- 평균 판단 시간, 재확인 수
- 최종 피로도, 최대 불안감, 최저 방향 확신도, 최대 시간 압박
- 장애물별 이벤트 로그
- 현재 유형의 관찰 포인트
- 결과 기반 자동 해석 문구

### 결과 이후 선택

- 다른 이동약자 유형 체험
- 같은 유형 다시 체험
- 시작 화면으로 이동

## 5. 제외 기능

다음 활동은 워크북과 팀 활동으로 이미 수행하므로 시뮬레이터에서 제외한다.

- 성찰 문항 입력 및 저장
- 완료 기록 누적 저장
- 결과 교차 비교
- JSON·CSV 내보내기

## 6. 교육적 원칙

- 시간과 상태 수치를 사람의 능력 점수로 표현하지 않는다.
- 결과를 개인의 결함보다 이동 조건과 환경 장벽의 상호작용으로 해석한다.
- 결과 화면은 워크북 성찰과 팀 토의를 시작하기 위한 관찰 자료로 사용한다.
