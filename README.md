# 동행 Access Lab 3D — 상황인지·판단·수행 이벤트 v0.4.1

고등 7~12차시에서 사용하는 이동약자 3D 체험 시뮬레이터입니다. 기존 기본 3D 환경, 장애물 5종, 이동약자 유형 시스템 위에 장애물별 학습 이벤트를 적용했습니다.

## v0.4.1 시작 흐름 수정

앱 실행 시 시작 안내와 이동약자 유형 선택을 먼저 표시하고, 체험자가 실제로 이동한 뒤 첫 장애물 이벤트가 열립니다.

## v0.4 구현 범위

- 장애물 접근 시 이벤트 자동 시작
- 접근 → 상황인지 → 판단 → 동작수행 → 결과 5단계 상태 머신
- 장애물별 판단 선택지 3개
- 안전 중심·위험 가능·재확인 선택
- 숫자 1·2·3, Enter/E, F 키 지원
- 이동약자 유형별 상황인지 문구와 판단 선택지
- 수행 중 충돌·차단·소요 시간 기록
- 장애물별 이벤트 결과 카드
- 최종 결과의 안전 판단·재확인·평균 판단 시간·이벤트 로그

## 이전 단계 유지

- 학교 정문 → 보도 → 횡단보도 → 버스정류장 3D 환경
- 장애물 O-01~O-05
- P-00 비교 기준
- P-01 수동 휠체어 사용자
- P-02 고령 보행자·지팡이 사용자
- P-03 시각장애인·흰지팡이 사용자
- 유형별 카메라 높이·속도·폭·반응 지연·장애물 규칙

## 실행

```bash
npm ci
npm run dev
```

브라우저에서 Vite가 안내하는 로컬 주소를 엽니다.

## 빌드

```bash
npm run build
npm run preview
```

## 주요 조작

- WASD / 방향키: 이동
- 마우스: 시점 회전
- Enter / E: 이벤트 다음 단계
- 1 / 2 / 3: 판단 선택
- F: 시각장애 유형의 정보 확인·지팡이 탐지
- ESC: 일시정지
- R: 현재 유형으로 처음부터

## 문서

- `EVENT_SYSTEM.md`: 이벤트 상태·판정 규칙
- `PERSONAS.md`: 이동약자 유형 규칙
- `OBSTACLES.md`: 장애물 규칙
- `V0.4_TEST_CHECKLIST.md`: 기능 검수 목록
- `BUILD_REPORT.md`: 빌드 결과

## 아직 포함하지 않은 기능

- 화면에 보이는 흰지팡이 3D 오브젝트와 애니메이션
- 시야 제한·고대비·블러 등 시각 효과
- 피로도·불안감·방향 확신도·시간 압박 HUD
- 유형 비교 결과 저장 및 CSV/JSON 내보내기
- 동료 평가와 교차 체험 화면

## 교육적 고지

이 시뮬레이터는 장애인의 실제 경험 전체를 재현하지 않습니다. 특정 이동 조건과 환경 장벽의 상호작용을 구조적으로 관찰하기 위한 교육용 모델입니다.
