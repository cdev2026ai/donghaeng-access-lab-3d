# 동행 Access Lab 3D — 이동약자 유형 시스템

고등 7~12차시에서 사용할 이동약자 3D 체험 시뮬레이터의 **6단계 이동약자 유형 시스템 적용 버전(v0.3)**입니다.

## 이번 버전에서 추가된 기능

### 4개 체험 프로필

- P-00 비교 기준 보행자
- P-01 수동 휠체어 사용자
- P-02 고령 보행자·지팡이 사용자
- P-03 시각장애인·흰지팡이 사용자

### 프로필별 차이

- 카메라 높이
- 기본 이동 속도
- 좌우 이동과 마우스 회전 민감도
- 사용자 충돌 폭
- 입력 반응 지연
- 최대 통과 단차
- 경사 구간 속도
- 장애물별 상황인지·상황판단·동작수행 문구

### 장애물별 유형 반응

- O-01 높은 단차
  - 비교 기준·고령 보행: 통과 가능 또는 감속
  - 휠체어·시각장애: 정면 통과 차단 후 경사로 우회
- O-02 가파른 경사
  - 유형별 경사 속도 배율 적용
- O-03 불법 주차 차량
  - 동적 사용자 폭으로 좁은 통로 통과 여부 변경
- O-04 파손 점자블록
  - 유형별 감속 차이
  - 시각장애 모드에서 방향 편차 발생
- O-05 볼라드·입간판
  - 휠체어 폭과 회전 조건에 따른 재진입

### 시각장애 모드 전용 기능

- `F` 키 지팡이 탐지
- 1.2m 범위의 장애물 접촉 정보
- 점자블록 단절 구간의 방향 편차 기록
- 결과 화면에 지팡이 탐지 횟수와 방향 편차 표시

## 기존 기능 유지

- 학교 정문·보도·횡단보도·버스정류장 기본 환경
- 장애물 5종과 충돌·감속·우회
- WASD·방향키 이동과 마우스 시점
- 장애물 진행 상태
- 보행 신호와 목적지 판정
- 저사양 모드·상황 안내·화면 흔들림 설정
- GitHub Pages 자동 배포

## 실행

Node.js 22 이상을 권장합니다.

```bash
npm ci
npm run dev
```

프로덕션 빌드:

```bash
npm run build
npm run preview
```

## 조작

- `W A S D` 또는 방향키: 이동
- 마우스: 시점 회전
- `F`: 시각장애 모드 지팡이 탐지
- `ESC`: 일시정지
- `R`: 현재 유형으로 처음부터

## 체험 권장 순서

1. 비교 기준 보행자로 전체 경로 완주
2. 수동 휠체어 사용자로 단차·주차 차량·볼라드 통과 여부 비교
3. 고령 보행자로 입력 반응, 경사 속도, 소요 시간 비교
4. 시각장애인 모드에서 F 키 탐지와 점자블록 단절 구간 확인
5. 결과 화면의 시간·충돌·차단·탐지·방향 편차 비교
