# 동행 Access Lab 3D — 이동 보조 앱 사전 체험 v0.9.2

학생들이 「이동약자의 이동 보조 앱」 프로젝트를 시작하기 전에 같은 경로를 이동약자 유형별로 직접 체험하고, 앱이 제공해야 할 정보와 기능을 발견하도록 돕는 교육용 3D 시뮬레이터입니다.

이 버전은 기획 문서 v2.0의 핵심 개정 사항을 코드에 반영했습니다.

- 장애물별 퀴즈·모달 중심 흐름 → 끊김이 적은 연속 이동 체험
- 횡단보도·보행 신호·차량 흐름을 핵심 장면으로 구현
- 앱 도움 없음/OFF와 앱 도움 있음/ON 조건 분리
- 유형별 횡단 속도·반응·정렬·정보 접근 제약 적용
- 성공 점수 대신 대기·출발·횡단·경로 조정·정보 부족을 관찰
- 성찰·교차 비교·저장·내보내기는 워크북과 팀 활동으로 분리

## v0.9.2 핵심 보완

- 3D 보행 신호등 본체에 적색·녹색 사람 모양과 남은 시간 표시
- 종료 임박 5초 점멸 표시
- HUD와 신호등 오브젝트의 연속 카운트다운 동기화
- 고령 보행자 횡단 속도와 출발 지연을 별도로 적용
- 12m 횡단보도와 17초 보행 시간 조건에서 P-02가 한 주기 안에 횡단하기 어려운 체험 구현
- 앱 ON에서 신호 연장 또는 보행 지원 필요 안내
- 신호 종료 후 횡단자가 남아 있으면 차량 안전 정지 유지


## 체험 흐름

```text
체험 목적 안내
→ 이동약자 유형 선택
→ 앱 도움 없음/있음 선택
→ 짧은 체험 안내
→ 학교 정문부터 버스정류장까지 연속 이동
→ 횡단보도 신호 대기·출발·횡단
→ 관찰형 결과 확인
→ 같은 조건에서 앱 도움 상태를 바꾸어 재체험
→ 워크북에서 앱 요구사항 정리
```

## 이동약자 유형

- `P-00` 비교 기준 보행자
- `P-01` 수동 휠체어 사용자
- `P-02` 고령 보행자·지팡이 사용자
- `P-03` 시각장애인·흰지팡이 사용자

각 유형은 카메라 높이, 이동 속도, 충돌 반경, 반응 지연, 횡단 정렬 시간, 단차 통과 조건, 시야·방향 정보 조건이 다릅니다.

## 연속 경로

```text
학교 정문
→ 보도 진입·단차와 경사로
→ 횡단보도 대기선
→ 보행 신호 확인
→ 12m 횡단보도
→ 반대편 경사로
→ 불법 주차 차량·파손 점자블록·볼라드
→ 버스정류장
```

장애물마다 전체 화면을 멈추는 선택 창은 기본 체험에서 사용하지 않습니다. 사용자의 정지, 시점 회전, 대기, 우회, 경사로 정렬, 지팡이 탐지, 실제 이동 경로를 관찰 기록으로 남깁니다.

## 횡단보도·신호

보행 신호 상태 머신:

```text
INIT_SAFE
→ RED_WAIT 8초
→ GREEN_START 1초
→ GREEN_ACTIVE 17초
→ FLASH_WARNING 6초
→ ALL_STOP 2초
→ RED_WAIT
```

- 보행 가능 신호 동안 차량이 정지선 앞에서 정지
- 사용자가 위험하게 진입하면 차량 비상 정지 및 기록
- 횡단 시작 시점, 대기 시간, 시작 시 남은 시간, 횡단 시간, 신호 종료 전 완료 여부 기록
- 예상 횡단 시간과 남은 시간을 비교해 앱 안내 생성

## 이동 보조 앱 OFF / ON

### 앱 도움 없음

- 환경에 존재하는 표지·신호·점자·음향 정보만 사용
- 자동 거리·방향·경사로·예상 시간·행동 권고 없음
- 체험 후 부족했거나 확인하기 어려웠던 정보를 결과에 정리

### 앱 도움 있음

비모달 패널과 선택적 TTS로 다음을 제공합니다.

- 보행 신호 상태와 잔여 시간
- 현재 속도를 반영한 예상 횡단 시간과 시간 여유
- 출발·대기·주의 권고
- 횡단 진입·이탈 경사로 방향과 거리
- 장애물 종류, 거리, 통과 조건, 우회 방향
- 시각장애 유형의 횡단 방향과 남은 거리

## 결과 화면

점수나 성공률 대신 다음 관찰값을 표시합니다.

- 신호 대기 시간
- 출발 반응 시간
- 출발 시 남은 신호 시간
- 횡단 소요 시간
- 신호 종료 전 완료 여부
- 경사로·방향 정렬 횟수
- 위험 진입과 차량 비상 정지
- 멈춤, 경로 재조정, 이동 차단·충돌
- 부족했던 정보
- 이동 보조 앱이 제공해야 할 정보

성찰 입력, 결과 누적 저장, 교차 비교, JSON·CSV 내보내기는 포함하지 않습니다.

## 분석 모드

강사·개발자용 `분석` 패널에서는 이동을 중단하고 다음을 확인할 수 있습니다.

- 상황인지·상황판단·동작수행 상태
- 신호 상태와 예상 횡단 시간
- 이동 보조 앱의 Input–Process–Output
- 현재 위치, 차단, 경로 조정 기록

기본 학생 체험에서는 이 패널을 열지 않아도 됩니다.

## 조작

- `WASD` / 방향키: 이동
- 마우스: 시점 회전
- `F`: P-03 흰지팡이 탐지

- 상단 `분석`: 강사·개발자용 상태 확인

## 교육적 고지

이 시뮬레이터는 모든 장애인과 이동약자의 실제 경험을 동일하게 재현하지 않습니다. 체험 결과는 개인 능력을 평가하거나 장애를 단순화하기 위한 점수가 아니라, 환경 조건과 정보 제공 방식이 이동에 미치는 영향을 분석하고 이동 보조 앱의 요구사항을 찾기 위한 교육 자료입니다.
