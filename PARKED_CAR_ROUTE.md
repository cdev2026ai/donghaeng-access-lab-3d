# v0.9.6 목적지 앞 장애물 재배치 적용 수정

## 문제 원인

v0.9.5에서 볼라드·입간판·점자블록의 위치 변경을 코드에 반영했지만, 덮어쓰기 패키지에 `src/main.ts`가 포함되지 않아 사용자가 기존 저장소에 적용했을 때 실제 화면이 바뀌지 않을 수 있었다.

## 수정 내용

### O-04 점자블록

- 횡단보도 북측 신호등 이후부터 목적지 방향으로 이어지도록 배치
- 불법 주차 차량 구간 이후에도 유도선이 연결되는 흐름 유지
- 중간에 훼손·단절 구간을 남겨 실제 환경의 문제를 관찰할 수 있도록 함

### O-05 볼라드·입간판

- 버스정류장 앞 통로로 이동
- 목적지에 도착하기 직전 통로 폭과 진입 각도를 제한하는 장벽으로 작동

### 권장 경로

```text
횡단보도 이탈
→ 불법 주차 차량 옆 좁은 통로
→ 점자블록 유도선
→ 훼손된 점자블록 구간
→ 목적지 앞 볼라드·입간판
→ 버스정류장 도착
```

## 테스트 포인트

- 덮어쓰기 후 `src/main.ts`가 변경되었는지 확인
- O-04 라벨이 신호등 이후 목적지 방향의 점자블록 구간에 있는지 확인
- O-05 라벨이 버스정류장 앞에 있는지 확인
- 청록색 권장 경로가 O-03 → O-04 → O-05로 이어지는지 확인
